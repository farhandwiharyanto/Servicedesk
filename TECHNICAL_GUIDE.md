# Technical Migration Documentation: Zoho Manage ServiceDesk

This document outlines the architectural shift from a monolithic Next.js (Prisma) application to a decoupled **Laravel 11 API Backend** and **Next.js 16 Frontend**.

## 1. Architectural Overview

| Component | Technology | Role |
| :--- | :--- | :--- |
| **Backend** | Laravel 11 | API, Business Logic, Data Persistence, AI Engine |
| **Frontend** | Next.js 16 | User Interface, Portal Navigation, State Management |
| **Database** | PostgreSQL | Relational Storage with UUID primary keys |
| **Auth** | Laravel Sanctum | API Token-based authentication |

## 2. Database Layer Changes

### UUID Implementation
All tables have been migrated to use **UUIDs** for increased security and scalability.
- **Trait**: `App\Traits\HasUuids` is applied to all 23 Eloquent models.
- **Pattern**: `protected $keyType = 'string'; public $incrementing = false;`

### Migrations
Legacy Prisma schemas were converted into standard Laravel migrations with strict foreign key constraints.
- **Reference**: `backend/database/migrations/`
- **Command**: `php artisan migrate:fresh` (Full sync)

## 3. API Reference

The following endpoints are now live and serving the frontend:

### Core Modules
- `GET    /api/lookups`: Fetches all metadata (Categories, Statuses, Priorities, Users) for dropdowns.
- `GET    /api/dashboard`: Aggregated stats and recent activity.
- `POST   /api/chatbot`: RAG-based AI assistant (Searches KB Solutions).

### Service Desk Modules
- `GET    /api/tickets`: List of all Requests and Problems.
- `GET    /api/tickets/{id}`: Detail view for a specific request.
- `POST   /api/tickets/request`: Create a new service request.
- `POST   /api/inbound-email`: Automated email-to-ticket creation endpoint.
- `GET    /api/assets`: Inventory list (Hardware/Software).
- `GET    /api/changes`: Change management list.
- `GET    /api/solutions`: Public Knowledge Base articles.

### Project & Finance
- `GET    /api/projects`: Project and Release management data.
- `GET    /api/finance`: Purchase Orders and Contracts.
- `GET    /api/maintenance`: Asset maintenance schedules.

## 4. Frontend Transformation

### Removal of Prisma
The node dependency `@prisma/client` and the `prisma/` directory have been removed from the frontend to ensure a clean decoupled boundary.

### API Integration Utility
A centralized fetcher was created to handle communication:
- **File**: `frontend/lib/api.ts`
- **Feature**: Supports automatic headers, error handling, and `x-api-key` validation.

### Page Migrations
The following portals were refactored to use the new API:
- IT/Dashboard (`/it`)
- Requests & Detail View (`/requests`, `/requests/[id]`)
- Asset Management (`/assets`)
- Problem & Change Management (`/problems`, `/changes`)
- CMDB, Finance, Maintenance, Solutions.

1. **Backend**: `cd backend && php artisan serve`
2. **Frontend**: `cd frontend && npm run dev`
3. **Database**: Managed via `docker-compose.yml` on port **5433**.
